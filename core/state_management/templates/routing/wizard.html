{# 規則名稱 #}

{# 描述 #}

選填。詳細說明此規則的用途、應用場景與特殊注意事項。

{# 優先級 #}

當多個規則同時匹配時，數值越高的規則優先執行。建議緊急任務使用高優先級。

{# 分析方法與配置 #}

選擇要使用的分析演算法。系統將自動載入該方法的可用配置選項。

選擇該分析方法的具體執行參數配置，如模型版本、閾值設定等。

{# MongoDB instance #}

指定要使用的 MongoDB 資料庫instance名稱。保持預設值 "default" 即可使用主要資料庫。

{# JSON 篩選條件 #}

使用 JSON 格式定義資料匹配條件。系統會將符合這些條件的資料路由到此分析方法。 支援精確匹配、列表匹配 (IN 操作) 和複雜條件 ($eq, $ne, $gt, $gte, $lt, $lte, $in, $nin)。
範例： {"dataset_UUID": "mimii_batch_upload", "label": ["normal", "abnormal"]}

格式正確
